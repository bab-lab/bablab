Section 5 GitHub

  1. Download GitHub Desktop

  1. Create new repository on GitHub Desktop

-Make sure to select the correct parent folder

  1. Initialize the repository
  • If you press command + shift + . you can see the hidden git files

  1. Add you files to the folder

  2. Open back to GitHub Desktop

  • The blue dot means there are changes in that folder to commit

  1. Publish the repository (this sets up the repository online - choose which organization it should go to)

  2. Create the first commit (add a comment)

  • Committing saves the changes to your local git (your local computer record of changes)(local changes)
  1. Push the repository
  • Push the changes that git has catalogued to the online GitHub
  1. Go to GitHub online and you can see your repository with all of your files and latest commits
  • Important to note that the README file will display as the main page for the repository

  1. If you click on the commits button, you can browse your entire history of commits, explore the file, and download the version from any point in it’s history

  2. Adding a .gitignore file will allow you to “skip” over certain types of files that you don’t want to commit

  3. You can change the name of the root folder on your local directory - just be sure to use the “locate” function in GitHub Desktop to locate it
